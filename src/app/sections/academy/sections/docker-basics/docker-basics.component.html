Inhalt
Eine sehr hilfreiche Software im Bereich des Backends ist Docker. In diesem Abschnitt bringen wie Dir bei, was Container und Images sind und wofür sie verwendet werden.
Da für die kommende Sektion nur ein kleiner Teil von Docker verwendet wird, wollen wir hier auch noch nicht zu tief in das Thema eintauchen. Wir klären hier nur die Basics.
Was ist Docker
Docker bietet die Möglichkeit, Software so zu verpacken, dass sie in jeder Umgebung laufen kann. Unterteilt wird das Ganze in drei Teile: Dockerfile, Image und Container.
Ein Dockerfile ist ein Entwurf, um Images zu bauen. Ein Image ist ein Template, um Container laufen zu lassen. Ein Container ist ein laufender Prozess.
Lässt Du zum Beispiel auf Deinem Laptop oder PC eine Anwendung laufen, die einwandfrei funktioniert, heißt das nicht, dass sie auf allen Geräten funktioniert. Gründe dafür können z. B. unterschiedliche Versionen von Paketen, aber auch andere Betriebssysteme sein. Durch Docker kann solchen Fehler vorgebeugt werden, da die Anwendung nun immer in derselben Umgebung läuft.
