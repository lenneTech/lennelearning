Inhalt
Eine sehr hilfreiche Software im Bereich des Backends ist Docker. Was sind Container und Images und wofür werden sie verwendet? Das bringen wir dir in diesem Abschnitt bei.
Da für die kommende Sektion nur ein Kleinteil von Docker verwendet wird, wollen wir hier auch noch nicht zu viel abarbeiten. Wir klären nur die Basics.
Was ist Docker
Docker ist eine Möglichkeit, Software so zu verpacken, dass sie in jeder Umgebung laufen kann. Unterteilt wird dies in drei Teile: Dockerfile, Image und Container.
Ein Dockerfile ist ein Entwurf, um Images zu bauen. Ein Image ist ein Template, um Container laufen zu lassen. Ein Container ist ein laufender Prozess.
Lässt du zum Beispiel auf deinem Laptop oder PC eine Anwendung laufen, die einwandfrei funktioniert, heißt das nicht, dass sie auf allen Geräten funktioniert. Da kommen unterschiedliche Versionen von Paketen, aber auch andere Betriebssysteme als Grund für auf. Durch Docker können solche Fehler vorgebeugt werden, da die Anwendung nun in derselben Umgebung läuft.
