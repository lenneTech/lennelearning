Inhalt
Jetzt bist du soweit, dass du große Projekte auf die Beine stellen könntest. Da ist es häufig sinnvoll in einem Team zu arbeiten und gemeinsam am Code zu werkeln. Spätestens im Beruf in einem IT-Team wirst du darauf stoßen, mit mehreren Entwicklern am gleichen Code zu sitzen. Dabei können viele Probleme auftreten: Meherere arbeiten am gleichen Problem und damit verliert man Zeit oder du hast noch gar nicht den neuesten Stand des Projekts. Um diese Probleme zu vermeiden nutzen wir Git und die damit kommenden Funktionen. In den nächsten Abschnitten erfährst du unter anderem, wie du Branches (Abzweigungen) des Hauptcodes erstellst, damit arbeitest und diese später wieder zum Hauptstrang hinzufügst.

Gästebuch
Setze die gelernten Inhalte in die Praxis um. Deine Aufgabe ist es, einen Eintrag in unserem Gästebuch zu hinterlassen. Hier eine kurze Anleitung:
Klone das Github-Repository vom Gästebuch
Erstelle einen neuen Branch mit dem Namen student/[DeinName] .
Schreibe in das Gästebuch einen Eintrag mit deinem Namen.
Führe git add . aus.
Führe git commit -m "rezension von [DeinName]" aus.
Bevor du pushst solltest du schauen, ob sich, während du gearbeitet hast, etwas an dem Hauptstrang geändert hat. Es könnte zum Beispiel sein, dass ein anderes Akademiemitglied gleichzeitig eine Rezension schreibt. Dafür machst du folgendes:
Führe git checkout main aus, um in den Hauptzweig zu wechseln.
Führe git fetch aus, um zu prüfen, ob Daten, die nicht in deinem Ordner sind, auf Github liegen.
Führe git pull aus, um die Daten, die nicht in deinem Ordner sind, von Github herunterzuladen.
Danach gehst du mit git checkout student/[DeinName] in deinen Branch
Und checkst mit git rebase main ob deine Änderungen ohne Konflikte bestehen bleiben können.
Wenn hier Konflikte auftreten, solltest du diese beheben.
Danach mit git push das ganze in Github pushen.
Jetzt gehst du in Github auf das Repository, gehst in den Tab Pull requests und forderst ein Review von deinem Tutor an.
Wenn alles in Ordnung ist, bekommst du eine Bestätigungsmail ansonsten eine Mail mit Änderungswünschen - das alles siehst du auch auf Github.
Wenn Änderungswünsche vorliegen, solltest du diese vollziehen und ein neues Review fordern.
Wenn alles bestätigt ist und gemerged wurde, bist du fertig.
