Projekt
Die Privatbrauerei Lennetaler braucht eine neue Website. Zunächst soll die Website rein gestalterisch mit Bootstrap umgesetzt werden. Folgende Anforderungen werden gestellt:
Eine Navbar mit Logo und Name, die Navigationsmöglichkeiten zur „Startseite“, „Shop“ und „Kontakt“, eine Suchzeile, um die Webseiten (nicht funktional) nach Inhalten durchsuchen zu können und einen Warenkorb, der sich ausklappen lässt und ausgewählte Produkte beinhaltet.
Eine Startseite, die den Nutzer mit einem Begrüßungstext Willkommen heißt (nutze dazu z. B. einen „Jumbotron“).
Eine Shopseite mit einer Diashow von unternehmensspezifischen Bildern on top und einer Auflistung der verschiedenen Biersorten (Lennetaler „Eisenglanz“, „Bernstein“ und „Goldberg“) mithilfe von „Cards“. Die einzelnen Karten haben ein Bild, den Produktnamen und einen „Mehr Anzeigen“-Button, der die Kartendetails ausfahren lässt. „Goldberg“ ist neu im Sortiment und soll dementsprechend gekennzeichnet werden.
Eine Kontaktseite, die einerseits die Kontaktdaten von Lennetaler mit einer „List Group“ anzeigt und zudem ein Kontaktformular zur Buchung von Brauereibesichtigungen (E-Mail, Name, Vorname, Teilnehmeranzahl, Nachricht) beinhaltet. Nutze dafür die Bootstrapvalidierung.

Was ist Responsive Webdesign
Responsive Design
Responsive Webdesign ist durch den Zugriff auf Websites über mobile Endgeräte entstanden. Durch den mobilen Zugang zu Website ist es nötig, diese nicht nur für Desktop-Ansichten, sondern auch für die mobile Nutzung zu entwickeln und zu optimieren. Geschieht das nicht, kommt es zu einer Reihe von Darstellungsproblemen, wodurch die User Experience enorm beeinträchtigt wird. Responsive Webdesign kann als Ansatz verstanden werden, Webdesign so zu entwickeln und zu gestalten, dass die Darstellung flexibel auf die Eigenschaften und Größenverhältnisse der verwendeten Bildschirme und Devices angepasst wird. Das Ziel eines Responsive Webdesign ist es, die Benutzerfreundlichkeit über alle Instanzen hinweg zu gewährleisten und Nutzer:innen unabhängig vom verwendeten Endgerät uneingeschränkten Zugang zu einer Website zu ermöglichen.
Warum Responsive UX Webdesign?
Warum UX Design?
Responsive Webdesign ist heutzutage ein fester Bestandteil in der App- und Webentwicklung und wichtig, um die User Experience nicht zu beeinträchtigen. Ein Webdesign sollte nach Möglichkeit auf allen Endgeräten ein einheitliches Bild schaffen – und das unabhängig davon, ob eine Website über ein Handy, Tablet oder klassisch über einen PC geöffnet wird.

Responsive Design mit Bootstrap
Responsive Bootstrap
Wir schauen uns nun vor allem die Begrifflichkeiten Breakpoints und Bootstrap an. Durch Bootstrap bekommen wir die Möglichkeit, mit dem Zuweisen einer Klasse, mehrere vorgefertigte Styles abzurufen und auf den jeweiligen HTML-Tag anzuwenden. Darüber hinaus bietet Bootstrap sogenannte Breakpoints. Diese verändern die Darstellung der einzelnen Inhalte der Webseite je nach Bildschirmgröße.

Zum Beispiel:
Eine Bildergalerie soll vier Spalten zur Darstellung haben. Dies ist mit Bootstrap schnell gelöst, indem man die col-Klasse verwendet und auf alle vier Spalten anwendet: class:"col-3". In der mobilen Ansicht wird dieser Inhalt jedoch sehr unschön aussehen, da nun immer noch vier Bilder in der selben Reihe vorliegen. Hier kommen die Breakpoints zum Einsatz: Nun verwenden wir class:"col-md-3 col-12" wodurch bei allen Displays kleiner als 768px (entspricht md) für jede Spalte eine Reihe erstellt wird. Für Displays ≥ 768px sind es vier Spalten. Eventuell überlappen die Bilder direkt am Breakpoint noch. Das kann behoben werden, indem man an weiteren Stellen Breakpoints hinzufügt: class:"col-lg-3 col-md-4 col-12". Nun sind in einer Reihe vier Spalten auf ganz großen Displays, drei Spalten auf etwas kleineren und eine Spalte auf mobiler Ansicht. Das ist nur ein kleiner Teil der Möglichkeiten, die Bootstrap im Bereich des Responsive Design mit sich bringt. Für mehr Informationen darüber ist die Dokumentation von Bootstrap zu empfehlen.
